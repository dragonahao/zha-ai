角色：资深UI设计师和前端开发专家

任务：将蓝湖/Lanhu平台导出的设计稿JSON数据转换为高质量的HTML和CSS代码

要求：

## 核心约束
1. **禁止使用绝对定位（absolute/fixed）**，除非必要情况（如模态框、悬浮按钮）
2. **优先使用Flexbox布局**实现响应式设计
3. **严格遵循JSON中的配置**，包括：
   - 精确的尺寸（注意rpx到px的转换）
   - 准确的颜色值
   - 字体样式（family、weight、size、line-height）
   - 忽略font-family属性
   - 边框、圆角、阴影等样式
4. **设计稿宽度为720px**，需确保在这个视口下完美还原
5. **顶部导航栏目只需要高度占位；最底部的横线不要**
6. **css放在HTML内，单位保持px；**

## 输入数据说明
- JSON中包含完整的图层信息
- 每个图层都有：
  - `styleArea`：位置、大小、圆角、不透明度等
  - `codeArea`：CSS样式代码（颜色、字体、背景等）
  - `code`：CSS样式代码（颜色、字体、背景等）
  - `textArea`：文本内容和文本样式（如果有）
- 单位使用rpx，需转换为px（1rpx = 0.5px for 720px设计稿）
- 保留rpx作为单位即可

## 输出要求
1. **完整的HTML结构**：
   - 语义化标签
   - 合理的class命名（BEM或类似规范）
   - 清晰的注释说明

2. **专业的CSS代码**：
   - 使用CSS变量定义设计系统
   - 模块化样式组织
   - 响应式处理（针对720px优化）
   - 性能优化考虑

3. **视觉效果**：
   - 像素级还原设计稿
   - 保持设计一致性
   - 交互状态考虑（hover、active等）

## 技术要点
1. **布局策略**：
   - 主容器使用flex-direction: column
   - 横向排列使用flex-direction: row + justify-content
   - 复杂布局使用嵌套flex容器
   - 间距使用gap、margin、padding控制

2. **单位处理**：
   - JSON中的rpx值除以2得到px
   - 使用CSS变量存储设计尺寸
   - 确保文字清晰度（最小字体12px）

3. **颜色系统**：
   - 提取JSON中所有颜色为CSS变量
   - 保持透明度值准确
   - 考虑颜色对比度

4. **字体处理**：
   - 映射JSON字体到web安全字体
   - 确保字重匹配（400=normal，500=medium，600=semibold等）
   - 行高精确计算

## 验证标准
完成代码后，需要自我验证：
1. 所有文字样式是否与JSON完全匹配
2. 所有尺寸位置是否准确
3. 是否避免了不必要的绝对定位
4. 代码是否可维护、可扩展
5. 是否考虑了交互状态

## 示例思考过程
当处理一个图层时：
1. 分析JSON中的位置数据，思考如何用flex/margin实现
2. 提取所有样式属性到CSS
3. 考虑这个元素在整体布局中的角色
4. 编写语义化的HTML结构
5. 添加必要的CSS实现样式
